# Python ビッグデータ処理 実践チートシート

巨大なデータ（数百万行〜テラバイト級）をPythonで処理する際、単純な `pandas.read_csv()` では「Out of Memory（メモリ不足）」でシステムがフリーズします。
本ドキュメントは、その壁を突破し、実務で爆速・安定稼働させるための最強の手法とコードをまとめたチートシートです。

---

## 基礎編：メモリの壁を越える2大アプローチ

巨大データを扱う場合、まずは以下のどちらか（あるいは両方）のアプローチで「メモリのパンク」を防ぎます。

### 1. 遅延評価（Lazy Evaluation） / Polars
処理の指示だけを先に蓄積し、最後に「一番賢いルート」を計算して一気に実行する手法です。

* **ミソ（クエリオプティマイザの最適化）:** 人間が書いた非効率なコードを、システムが実行直前に「不要な列は読まない」「不要な行は事前に捨てる」「処理順序を組み替える」など、極限まで無駄を省いた設計図に書き換えてから処理します。
* **メリット:** 数千万行のCSVデータであっても、メモリ消費を最小限に抑えつつ、PCの全コアをフル活用（マルチスレッド）して一瞬で集計を完了させることができます。

```python
import polars as pl

# read_csvではなく、scan_csvで「遅延評価（読み込む準備だけ）」にする
query = pl.scan_csv("massive_data.csv")

# 処理の設計図を作る（この時点ではまだ計算されない）
plan = (
    query
    .filter(pl.col("店舗名") == "東京")  # 絞り込み
    .group_by("商品名")                 # グループ化
    .agg(pl.col("売上").sum().alias("総売上")) # 集計
)

# collect() で一気に最適化された計算を実行する
result_df = plan.collect()
print(result_df)
